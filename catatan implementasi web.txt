ADMIN	:
data barang ->
admin bisa tambah, update, delete.
tapi tidak bisa update nama barang karna primary key.

member -> 
admin bisa edit dan delete, tapi tidak bisa edit nama member.
untuk login member tergantung ada atau tidak nya dalam database member

beli -> 
admin hanya bisa lihat data tabel beli yang diinputkan member

supplier -> 
admin bisa tambah, update, delete.
tapi tidak bisa update nama supplier karna nama supplier merupakan primary key.

supply ->
admin bisa tambah, update, delete. 
tapi tidak bisa update nama supplier.

note : 
setiap masuk barang dari member yang di beli client, maka stok di data barang (admin dan member) berkurang.

setiap admin mensupply barang dari supplier, maka stok di data barang (admin dan member) bertambah.

untuk menambahkan supply, nama supplier harus sudah terdaftar di tabel supplier.

untuk mensupply barang, terlebih dahulu barang tersebut harus ada di tabel databarang, 
baru admin dapat tambahkan barang sesuai dengan siapa yg jadi supplier.

MEMBER	:
galery ->

beli ->

tapi tidak bisa update nama barang dan nama member karna keduanya merupakan foreign key.

kekurangan web ini :
1. stok tidak bertambah pada saat member melakukan delete yang artinya member membatalkan pesanan.
2. stok tidak berkurang pada saat admin melakukan delete yang artinya membatalkan barang yang di supply.
3. melakukan penambahan barang yg di supply dari supplier tidak bisa dilakukan jika nama barang tidak ada di tabel data barang.
	artinya harus menambahkan nama barang terlebih dahulu di tabel barang.
4. melakukan penambahan barang yg di supply dari supplier tidak bisa dilakukan jika nama supplier tidak ada di tabel supplier.
	artinya harus menambahkan nama supplier terlebih dahulu di tabel supplier.
